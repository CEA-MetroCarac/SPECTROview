# Purpose

This file defines rules for Python code generation by the SPECTROview AI Agent. These rules apply when the user explicitly requests Python scripts or code snippets.

---

# Instructions

These rules govern the quality and safety of any Python code generated by the agent.

---

# Rules

## Code Quality

- **Generate readable code.** Prioritise clarity over cleverness. A junior developer should be able to understand the code without deep context.
- **Use descriptive variable names.** Match variable names to the user's column names and DataFrame names where possible.
- **Add comments only when useful.** Do not add obvious comments like `# import pandas`. Add comments to explain non-obvious logic, algorithm choices, or SPECTROview-specific conventions.
- **Follow PEP 8.** Use 4-space indentation, snake_case for variables and functions, UPPER_CASE for constants.
- **Avoid duplicated code.** Extract repeated logic into helper functions.
- **Prefer reusable functions** over copy-pasted code blocks.

## pandas Best Practices

- Use vectorised operations and built-in pandas methods. Avoid row-by-row Python loops.
- Use `.query()` for filtering, `.groupby()` for aggregation, `.merge()` for joins.
- Use `.assign()` for adding new columns rather than direct assignment in complex pipelines.
- Prefer `pd.concat()` over `pd.DataFrame.append()` (deprecated).
- Always handle potential `NaN` values when computing statistics.

## Plotting Code

- Use `matplotlib.pyplot` directly. Do not use `seaborn`.
- Always assign figures and axes: `fig, ax = plt.subplots()`.
- Use `ax.set_xlabel()`, `ax.set_ylabel()` for axis labels.
- Save figures with `fig.savefig(path, dpi=150, bbox_inches='tight')`.
- Call `plt.close(fig)` after saving to release memory.

---

# Constraints

- **Never use `eval()`, `exec()`, `__import__()`, or `compile()`** under any circumstances.
- **Never write to user files** without explicit user instruction.
- **Never use `seaborn`** — use `matplotlib` for all plotting.
- Do not generate network requests (`requests`, `urllib`, `httpx`).
- Do not generate OS commands (`os.system`, `subprocess`, `shutil.rmtree`).
- If a task requires unsafe operations, use `action: "answer"` to explain the limitation.
